# 2026-05-22

## What I worked on

Added `npm run snapshot` — a Playwright script at `scripts/snapshot.js` that captures full-page PNGs of every route and archives them under `snapshots/YYYY-MM-DD_HHMM/`. Added a `generateSampleData` function that seeds 3 runners (Alice, Bob, Carol) with 10 × 1km intervals (5–6 min per lap, ~1 min rest) using a seeded LCG so output is deterministic.

## Decisions made

Used hash-history URLs (`/#/` and `/#/report`) to match the app's `createWebHashHistory` router, with `BASE_URL = 'http://localhost:5173/interval-coach'` matching the Vite dev server base. Followed the two-pass footer-bottom height measurement from the reference implementation to avoid `min-height: 100vh` inflation.

## What I learned

The reference script's `nextjs-portal` hide step is kept for parity but is a no-op in this Vite/Vue app. The report and home views both render from `runnerIntervals` (the simple `startMs`/`stopMs` model), not from `laps` or `events` — so mock data only needs to populate `timing.runnerIntervals`, not the full event log.

## Problems / friction

Hash-route `goto` calls in Playwright/Chrome are same-document navigations — the page doesn't reload, so Pinia never re-hydrates from the injected localStorage. Fixed by calling `page.reload()` after `goto` whenever a setup function is present.
